Here's what you built:

    View Fragment-based view tracking

    Each time this wizard changes location.hash, the RUM SDK records a new View event. No page reload needed — the SDK detects hash changes automatically.

    In RUM › Explorer, filter to view events and group by @view.url_hash to see per-step metrics: load time, action count, frustration signals, and drop-off.

    source:browser service:datadog-demo-app
@type:view
| stats count() by @view.url_hash

    Each hash maps to a named step:

    #crust→ #sauce→ #cheese→ #toppings→ #size→ #results
    Funnel Conversion funnel by view

    RUM Analytics has a native Funnel visualization. Define each step as a view filter on @view.url_hash to see the conversion rate and absolute drop-off between steps.

    Go to RUM › Analytics, set the visualization to Funnel, then add steps:

    Step 1  @view.url_hash:#crust
Step 2  @view.url_hash:#sauce
Step 3  @view.url_hash:#cheese
Step 4  @view.url_hash:#toppings
Step 5  @view.url_hash:#size
Step 6  @view.url_hash:#results

    The funnel shows the percentage of sessions that reached each step, making it easy to spot where users abandon the wizard.

    Metric Custom metrics from order actions

    When a user completes the wizard, this app fires a custom RUM action:

    DD_RUM.addAction('pizza_order_submitted', {
  pizza_order: {
    crust:    "Hand-Tossed",
    sauce:    "BBQ",
    cheese:   "Mozzarella",
    toppings: "Pepperoni",
    size:     "Large (16 in)"
  }
})

    In RUM › Generate Metrics, create a count metric from this action to track submission volume over time:

    Event type:  Action
Filter:      @action.target.name:pizza_order_submitted
Metric name: rum.pizza_order.submissions
Group by:    @context.pizza_order.crust
             @context.pizza_order.size
             @context.pizza_order.sauce

    The resulting metric can be used in dashboards, monitors, and SLOs — for example, alerting if submission volume drops below a threshold, or graphing a breakdown of popular crust types over time.
